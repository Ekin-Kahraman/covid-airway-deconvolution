# Security policy

## Supported versions

Security and correctness fixes are made on the default branch.

## Reporting a vulnerability

This repository downloads public expression data and processes local model artefacts. If you find unsafe file handling, credential leakage, model-loading risk, or dependency supply-chain issues, do not open a public issue.

Email: evk23umu@uea.ac.uk
Subject: `covid-airway-deconvolution security: <short description>`

Please include:
- Reproduction steps
- Python and PyTorch versions
- Input file type and source
- Expected impact

I aim to respond within 7 days.

## Data handling

Do not attach private clinical data, patient metadata, or restricted-expression matrices to public issues. Use public accession IDs, synthetic data, or redacted examples.
